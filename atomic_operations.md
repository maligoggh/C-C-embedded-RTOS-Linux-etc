## В 2 предложениях

**Атомарная операция** — неделимая. Либо выполняется целиком, либо не выполняется вообще. Никакой "половины".
### Аналогия

**Неатомарно:** снимаешь деньги в банкомате
1. Проверяешь баланс (1000 руб)
2. *(прерывание — жена звонит)*
3. Снимаешь 1000 руб
4. Записываешь новый баланс (0 руб)

Проблема: между шагом 1 и 3 кто-то другой тоже может снять деньги. Итог: минус на счёте.

**Атомарно:** снятие деньги — одна операция, нельзя прервать посередине.

## В коде: неатомарная операция

```c
// @learned reg |= (1 << 3) — это ТРИ операции, не одна!
reg |= (1 << 3);

// На самом деле:
uint32_t temp = reg;      // 1. Читаем регистр
temp = temp | (1 << 3);   // 2. Модифицируем
reg = temp;               // 3. Пишем обратно
```

**Проблема:** между шагом 1 и 3 может произойти прерывание, которое тоже изменит `reg`. Твои изменения затрут чужие.

## Сценарий ошибки

```c
// Основной код
reg |= (1 << 3);  // Читает reg = 0b0000

// Прерывание! (вклинивается посередине)
TIM2_IRQHandler() {
    reg |= (1 << 5);  // Тоже читает reg = 0b0000, пишет 0b100000
}

// Возврат в основной код
reg = 0b0000 | 0b1000;  // reg = 0b1000, бит 5 ПОТЕРЯН!
```

## Решение: атомарные операции

**Способ 1: Специальные регистры (STM32)**
```c
// BSRR — Bit Set Reset Register, одна запись = атомарно
GPIOA->BSRR = (1 << 5);        // Установить PA5 — нельзя прервать
GPIOA->BSRR = (1 << (5 + 16)); // Сбросить PA5 — нельзя прервать
```

**Способ 2: Запрет прерываний (критические секции)**
```c
// @pattern Критическая секция
__disable_irq();      // Запретить прерывания
reg |= (1 << 3);      // Безопасно, никто не вклинится
__enable_irq();       // Разрешить прерывания
```

**Способ 3: Атомарные инструкции процессора**
```c
// LDREX/STREX в ARM (Load-Exclusive / Store-Exclusive)
// __atomic_fetch_or в GCC
__atomic_fetch_or(&reg, (1 << 3), __ATOMIC_SEQ_CST);
```

## Проверка: атомарно или нет

| Операция | Атомарно? | Почему |
|----------|-----------|--------|
| `reg = 0xFF` | ✅ Да | Одна запись |
| `reg |= (1 << 3)` | ❌ Нет | Чтение-модификация-запись |
| `GPIO->BSRR = val` | ✅ Да | Специальный регистр |
| `reg++` | ❌ Нет | Чтение, инкремент, запись |
| `__atomic_fetch_add` | ✅ Да | Процессорная инструкция |
## Для embedded критично

- **Прерывания (ISR):** короткие, быстрые, но могут вклиниться в любой момент
- **Многопоточность (RTOS):** два потока лезут в один регистр
- **DMA:** железо пишет в память параллельно с CPU
