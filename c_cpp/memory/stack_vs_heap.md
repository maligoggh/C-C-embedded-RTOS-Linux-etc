# Стек vs Куча в Embedded

## В 2 предложениях
Стек — быстрое выделение с известным размером, но ограничен. Куча — гибкая, но медленная и фрагментируется. В embedded стек предпочтительнее, куча — под вопросом.

## Почему это критично для embedded
- Предсказуемость: стек O(1), куча O(n) + неопределенность фрагментации
- Безопасность: переполнение стека = HardFault, утечка кучи = молчание до отказа
- Ресурсы: типично 4-8KB стека vs вся оставшаяся SRAM

## Память в Cortex-M (типичная карта)

┌─────────────────────┐ ← 0x2000_0000 (SRAM start)
│ .text (in Flash)    │
├─────────────────────┤
│ .data (инициализир.) │
│ .bss (нулевая)       │
├─────────────────────┤ ← _end (начало кучи)
│ Куча (растёт вверх)  │
│         ↑           │
│         ↓           │
│ Стек (растёт вниз)   │
├─────────────────────┤ ← _estack (начало стека)
│                     │
└─────────────────────┘ ← 0x2000_0000 + SRAM_SIZE
